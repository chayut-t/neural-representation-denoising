# Decision Records

Short architecture/science decision records (ADRs) that make the plan executable. A default in
plan §3 changes only via a record here (plan §3 preamble). Each record: context, decision,
alternatives, consequences, status.

These records freeze the choices required by Gate P1 (plan §7 Phase 1, task 4) before any
confirmatory experiment. Where a record only ratifies a plan §3 default, it says so; where it
diverges (e.g. offline package install), it justifies the divergence.
